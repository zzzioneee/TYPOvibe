# Day 001 — 클래식 오르골 타이포그래피

## 컨셉

타이핑 한 자 한 자가 오르골의 레버를 돌리는 물리적 행위가 되어 클래식 선율을 완성합니다.

## 영감

- **J.S. Bach — Air on the G String** (BWV 1068, Suite No.3 in D Major)
- **Red Velvet — Feel My Rhythm**: G선상의 아리아를 샘플링한 K-pop. 클래식을 현대적으로 재해석한 사례에서 영감을 받음.

## 구현 과정

### 1. 소스 음악 준비

Red Velvet 'Feel My Rhythm'이 G선상의 아리아를 샘플링했기 때문에, 비교적 소스를 구하기 쉬운 클래식 원곡을 활용.

1. G선상의 아리아 원곡을 **MP3**로 확보
2. AI가 음 데이터를 인식할 수 있도록 MP3를 **MIDI로 변환**
   - 사용 도구: [OpenMusic.ai MIDI Editor](https://www.openmusic.ai/ko/midi-editor) (무료)
3. MIDI 파일에서 멜로디 노트 시퀀스를 추출하여 코드에 반영

### 2. MIDI → 코드 변환의 어려움

피아노 편곡 MIDI는 멜로디와 반주가 한 트랙에 섞여있어서 자동 추출이 어려웠음.

- 최고음 추출, 음역대 필터링, stepwise motion 추적 등 여러 알고리즘을 시도
- 결국 멜로디 음역대(F#4~A5)를 필터링하고, 직전 음과의 점프가 7반음 이내인 음만 선택하는 방식으로 추출
- 튀는 음(옥타브 이상 점프)은 수동으로 한 옥타브 내려서 보정

### 3. 오르골 사운드 합성

Web Audio API로 오르골 소리를 합성.

- **Oscillator**: Sine 파형 (오르골 특유의 맑은 소리)
- **배음 구조**: 기본음 + 2배음(0.2) + 3배음(0.12) + 4배음(0.04)
- **Envelope**: Attack 0.04s → Release 3.0s (건반을 누르는 순간 + 긴 여운)
- **멜로디 + 하성부**: 각 타이핑마다 멜로디 음 + 하성부 음을 동시 재생 (하성부는 50% 볼륨)

### 4. 타이포그래피 인터랙션

- **타이핑 = 연주**: 한 글자 입력할 때마다 멜로디의 다음 음이 재생
- **GSAP 둥실둥실**: 각 글자가 등장 후 `sine.inOut` 이징으로 위아래 부유 (글자마다 다른 duration)
- **노란 하이라이트 커서**: 현재 입력 중인 글자에 `#ffff00` 배경, 다음 글자로 넘어가면 해제
- **오르골 핀 시각화**: 하단에 멜로디 노트 수만큼 핀 배치, 음 재생 시 해당 핀이 활성화

### 5. 한글 IME 처리

한글 입력은 자음+모음 조합 과정이 있어서 별도 처리가 필요했음.

- 숨겨진 `<input>` 요소로 IME 조합을 처리
- `compositionupdate`로 조합 중인 글자를 실시간 미리보기
- `compositionend`로 확정된 글자를 화면에 추가 + 음 재생
- 스페이스/엔터 중복 입력 방지를 위한 `justComposed` 플래그

### 6. 비주얼 디자인

- **봄 느낌**: 흰 배경 + 사방 노란 그라데이션 테두리 (`box-shadow: inset`)
- **명조체**: 나눔명조 (클래식 + 오르골의 아날로그 감성)
- **텍스트 색상**: `#58595b` (차콜 그레이)
- **하이라이트**: `#ffff00` (퓨어 옐로우)
- **Ctrl+A**: 전체 선택 (노란 하이라이트) → Backspace로 전체 삭제

## 기술 스택

## 동료 공유용 요약

### 한 줄 요약
키보드를 치면 G선상의 아리아가 오르골처럼 한 음씩 재생되는 인터랙티브 타이포그래피.

### 왜 이걸 만들었나
레드벨벳 'Feel My Rhythm'이 바흐의 G선상의 아리아를 샘플링한 것처럼, 클래식 음악을 타이포그래피로 재해석해보고 싶었음. 오르골이 레버를 돌려 한 음씩 소리를 내는 것처럼, 타이핑이 곧 연주가 되는 경험을 만들었음.

### 작업 흐름
1. G선상의 아리아 MP3 → MIDI 변환 (openmusic.ai)
2. MIDI에서 멜로디 노트 추출 → 코드에 반영
3. Web Audio API로 오르골 음색 합성
4. 타이핑할 때마다 다음 음 재생 + 글자 둥실둥실 애니메이션
5. 봄 느낌의 흰 배경 + 노란 그라데이션 테두리 + 명조체

### 핵심 인터랙션
- **타이핑 = 오르골 연주**: 한 글자 = 한 음. 멜로디 + 하성부 동시 재생.
- **노란 하이라이트 커서**: 현재 입력 글자에 형광 노란 배경이 따라다님.
- **둥실둥실 부유**: 입력된 글자들이 각자 다른 리듬으로 떠다님 (GSAP).
- **오르골 핀 시각화**: 하단에 음 재생에 반응하는 핀 바 표시.

### 삽질 포인트
- MIDI에서 멜로디 자동 추출이 어려움 (피아노 편곡은 반주가 섞여있어서). 여러 알고리즘 시도 끝에 음역대 필터 + 점프 제한으로 해결.
- 한글 IME 처리가 까다로움. 영어는 keydown으로 바로 되지만, 한글은 compositionend를 써야 하고 스페이스/엔터 중복 방지도 필요.
