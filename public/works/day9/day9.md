# Day 9 — 러브버그 출몰 시기

**날짜**: 2026.04.28
**헤드라인**: 러브버그 출몰 시기가 돌아왔다?! 미리 방제 작업 나선 계양산
**출처**: [뉴닉](https://newneek.co/@newneek/article/40266)

---

## 컨셉

러브버그 출몰 시기가 돌아왔다는 뉴스를 타이포그래피로. LOVE(초록)와 BUG(노랑) 두 글자 덩어리가 **엉덩이를 맞댄 한 쌍**으로 날아다니는 모습. 실제 러브버그가 암수 한 쌍으로 붙어 다니는 특징을 글자 두 개의 쌍으로 번역. 클릭하면 튕기며 낙하하고 퇴치 수가 쌓임.

---

## 구현 내용

### 1. 글자 구성 (픽셀 폰트 + 원 집합)
- LOVE, BUG 각 알파벳을 5x5 픽셀 패턴으로 정의
- 각 픽셀을 10px 반경 원으로 치환, 간격 14px
- 원마다 개별 **노랑/초록 fill + 빨강 stroke** — 곤충 외피 느낌
- LOVE 색 `#03ff00`, BUG 색 `#ffff00`, 테두리 `#FF0000`, 배경 `#0076ff`(fallback)

### 2. 쌍 구조 (러브버그 암수 한 쌍)
- LOVE 컨테이너는 `rotate(-8°)`, 중심 (100, 50)
- BUG 컨테이너는 `translate(+200, +15)` + `rotate(+8°)`, 중심 (75, 50)
- 두 글자가 서로 반대로 기울어져 위아래로 살짝 겹치며 **엉덩이 맞대기** 실루엣
- 양쪽 머리(L, B)에 검정 더듬이 V자 + 끝 bulb
- LOVE의 L 머리 원(col=0, row=0) 중심에 더듬이가 딱 붙도록 좌표 튜닝

### 3. 비행 모션
- 각 쌍은 `dir: +1 / -1` 중 하나로 X축 횡단 이동만 (벌레가 화면을 가로지름)
- 속도: 1.4 + random × 1.0
- Y는 `sin(frame * yFreq) * yAmp` 로 느린 파도 + 날갯짓 미세 진동
- 몸체 S자 흔들림: `-8° + sin × ±1.2°` / `+8° - sin × ±1.2°` (서로 반대 위상)
- 화면 밖으로 나가면 반대편에서 리스폰

### 4. 클릭 분해
- 쌍의 bounding box 히트 테스트
- 클릭 시 LOVE는 **위-왼쪽**(`vx -6~-9, vy -8~-10`), BUG는 **위-오른쪽**으로 튕김
- 중력 `vy += 0.5`, 수평 감속 `vx *= 0.98`
- Y축 700px 밖으로 나가면 dead → 제거
- 퇴치 수 카운터 (우상단) 증가

### 5. 쌍 생성 로직
- 초기 2쌍, 최대 4쌍 유지
- 죽은 쌍 있으면 1.5초 후 보충 스폰
- 신규 쌍의 방향(dir)은 기존 쌍들과 균형 맞추기 (좌/우 섞임)

### 6. 배경 (블러 풀밭)
- `refer.jpg` (720×960 초록 모션블러 이미지)를 `#stage` 배경으로
- 그 위에 **`.bg-blur` 오버레이** — 같은 이미지를 `filter: blur(18px) + scale(1.1)`로 겹침
- 이중 레이어로 **원본 톤은 유지하면서 더 뭉개진 몽환감**

## 삽질 포인트

### "벌레처럼 보이는가" 에서 여러 번 실패
- 초반에 ㅅ+ㅗ 같은 한글 자소 기반 접근 시도 → 글자로 안 읽힘
- LOVE/BUG 라는 영문 구도로 바꾼 뒤에도 **더듬이 위치 / 몸통 / 날개 / 다리**를 붙였다 뗐다 반복
- 최종 해결: **글자 덩어리 두 개 + 기울기 + 더듬이**만 남기고 몸통/날개/다리는 전부 제거. 조형적으로 두 마리 한 쌍 실루엣이면 벌레로 읽힘.

### 날아다니는 느낌
- 잔떨림/랜덤 방향 전환으로 접근했다가 **제자리 왔다갔다만** 하는 상태
- 핵심 교훈: "날아다닌다 = X축 **일관 방향 횡단**이 메인". Y축 wave는 보조.
- `dir` 변수를 ±1로 고정하고 속도·방향을 **화면 가로 전체**를 횡단하는 식으로 바꾼 뒤에야 비행감 생김

### Sub-2 Beyond Limits (v2 폐기)
- 처음 day9 주제로 "러브버그"를 잡았다가 도중에 "Sub-2 Beyond Limits" 마라톤 주제로 전환 시도
- 등고선 / 수작업 path / SVG textPath / 런던 코스 재현 등 여러 방향 시도했으나 모두 퀄리티 불만족
- 실제 런던 마라톤 코스를 GPX 없이 수동 재현하려다 궤적이 엉뚱해짐
- 결국 v2 폐기하고 **v1 러브버그로 복귀**. v1 백업 파일은 정리

### 이미지 레퍼런스 해석 반복 실패
- 오늘 세션 내내 사용자가 첨부한 이미지의 색/형태를 여러 번 잘못 해석
- 앞 턴 레퍼런스와 현재 턴을 혼동하거나, 두 이미지 순서를 거꾸로 읽음
- `steering/rules/core/image-reference.md` 신설 — 이미지 첨부 시 색/형태/텍스트 명시적으로 짚고 확인받는 루틴 강제

## 다음에 다시 한다면

### 처음부터 이렇게 요청하면 좋을 것 같다

> "[LOVE/BUG 같은 2음절] 단어 두 개를 **픽셀 폰트(원 집합)** 로 만들고, 서로 **±α도 기울여 겹쳐 한 마리 곤충** 실루엣으로. 화면을 **X축 횡단**하며 날아다니는 쌍 여러 개, 클릭하면 글자 두 개가 반대로 튕기며 낙하. 배경은 [레퍼런스 이미지]."

- 곤충 = 몸통/날개/다리 없이 **글자 덩어리 + 더듬이 + 기울기**만으로 구현하는 방향을 처음부터 못 박으면 오버 디자인 왕복 제거 가능
- "날아다닌다"는 구현 전에 먼저 정의하기. **X축 일관 방향 횡단**이 메인이라는 점 명시
- 글자 자체를 디자인할 땐 Claude가 만든 원본 JSX 레퍼런스를 먼저 확보해두고 vanilla 포팅하는 게 빠름 (day8 패턴과 동일)

### 보완하면 좋을 것들

- **분해 파티클 개수 증가**: 지금은 LOVE 한 덩어리, BUG 한 덩어리로 2개만 튕겨나감. 각 원이 개별 파티클로 흩어지면 더 드라마틱할 듯
- **그림자**: 배경이 풀밭인데 벌레 바로 밑에 그림자 없어 붕 떠 보임. 타원 반투명 shadow 추가하면 "풀밭 위 비행" 더 자연스러움
- **소리 효과**: 붕붕 효과음 + 퇴치 시 "퍽" 효과음. day6의 Web Audio 패턴 재활용 가능
- **카운터 UX**: 숫자만 커지는 것보다 "벌레 잡기" 게임적 요소(숫자 pop / 콤보) 얹으면 재미 ↑
- **반응형**: viewBox/좌표가 화면 크기와 연동 안 되어 세로 화면에서 잘림 많음
- **배경 블러 강도 조정 인터랙션**: 지금 고정 18px인데 사용자 설정 변경 가능하게 하면 이터레이션 편해졌을 듯

### 시간 절약 팁

- **이미지 레퍼런스 첨부 시 색·형태·텍스트 3요소를 먼저 명시적으로 짚어 확인받기** (오늘 세션에서 steering 규칙으로 추가)
- **방향 전환은 신중히**: 중간에 "Sub-2"로 갈아타려다 v1까지 흔들렸다. 세션당 주제 하나 끝까지 가는 게 빠름
- **이미지 파일 vs 자체 재구현 먼저 선택**: 배경/이펙트 리소스는 이미지 그대로 쓸지 코드로 재현할지 초반에 정하면 왕복 없음
- **백업은 git 브랜치로**: `day9_v1.js` 같은 파일 백업보다 git branch 파서 버전 관리하는 게 깨끗

## 동료 공유용 요약

**왜 만들었나**: 러브버그 출몰 시즌 뉴스를 타이포그래피로. LOVE(초록)와 BUG(노랑) 두 글자가 **암수 한 쌍**처럼 기울여 겹쳐 날아다니는 쌍 아트웍.

**핵심 인터랙션**: 4쌍의 벌레가 풀밭 배경 위를 뽈뽈 횡단. 클릭하면 두 글자가 반대로 튕기며 낙하, 우상단 카운터 증가. 퇴치하면 새 쌍 스폰.

**특이점**: 몸통/날개/다리 같은 직설적 곤충 요소 전부 제거하고 **글자 덩어리 + 기울기 + 더듬이**만으로 곤충 실루엣 구축. 조형 최소주의.

**기술**: Canvas 2D + requestAnimationFrame + CSS blur 오버레이. 레터링 path는 Claude가 설계한 원본 JSX를 vanilla로 포팅.
