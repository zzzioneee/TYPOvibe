# Day 24 — Volatility Drag (음의 복리 시각화)

**2026.05.28 (수요일)**

## 출발

SK하이닉스 FOMO가 제대로 왔는데, 5월 27일 자로 2배 레버리지 ETF가 출시돼서 한번 알아보다가 나같은 애가 뭣모르고 덤비면 망하기 딱 좋겠다고 생각했음. 뉴닉에서 마침 레버리지 ETF의 위험성에 대한 기사를 읽고, "하루만 책임진다"는 개념 — 기초자산의 당일 수익률만 정확히 2배로 추종하고, 매일 리셋. 이걸 반복하면 횡보장에서 복리로 녹아내리는 구조. 국내 주식 가격 제한폭 ±30% 기준, 단일종목 레버리지 ETF는 이론적으로 하루 만에 최대 60% 손실 가능. FOMO로 사서 묵혀두면 필패하는 구조를 시각적으로 보여주고 싶었음.

## 컨셉

**스크롤 = 시간의 흐름**: 가로 스크롤로 150일간의 주가 변동을 탐색. 위쪽은 일반 주식(b), 아래쪽은 2배 레버리지(c). 같은 기초자산인데 시간이 지날수록 c가 처참하게 녹아내리는 걸 체감하게 만듦. 마지막에 c가 폭발하며 "이건 장기 투자 상품이 아니다. 단기 트레이딩용 독약이다"라는 메시지 전달.

## 구조

- 검정 배경 + 가로 스크롤 (10000px)
- 인트로: 중앙에서 a 노드 pop-in → b, c로 대칭 fork
- 타임라인: b(상단)와 c(하단) 차트가 동시에 전개
- b: 원 1개씩 스무스하게 이동, 떨림 없음
- c: 밀도 높은 원들이 뭉쳐서 떨림 + 상승 시 팽창, 하락 시 압착
- 튜브: 구간별 색상 (상승=빨강, 하락=파랑), quadraticCurveTo로 매끄러운 곡선
- 등락률 라벨: 3개 간격으로 당일 수익률 표시, 시간 지나면 fade out
- 엔딩: b, c 모두 끝까지 도달 → 0.5초 대기 → c 폭발 → 결과 텍스트

## 기술

### 주가 데이터 생성

150일간 랜덤 횡보 (`(Math.random()-0.5) * 0.10`) + 15일간 점진적 크래시. 레버리지는 매일 `1 + r*2`로 복리 계산.

```
stockPrices[i] = stockPrices[i-1] * (1 + dailyReturn)
levPrices[i] = levPrices[i-1] * (1 + dailyReturn * 2)
```

### 노드 밀도 (cNodesPerTick)

당일 수익률 크기에 비례해서 c 노드 밀도 결정. 변동성 클수록 밀도 높아짐.

### 튜브 렌더링 (drawTubeSegmented)

- 법선 벡터: 인접 노드 평균(prev→next)으로 스무딩 → 급격한 방향 전환에서도 일정한 두께
- 색상: `nodeData`(각 노드의 dataIdx)로 당일 수익률 기준 결정
- 곡선: midpoint 기반 quadraticCurveTo로 색 전환 지점에서도 매끄럽게

### 떨림 (Vibration)

`vibAmt = max(1.0, lossRatio * 3) * vibrationIntensity`

c 노드는 하락일에 1.8배, 상승일에 0.3배 떨림 적용. 손실이 커질수록 떨림 증가.

### 노드 보간

같은 틱 내 여러 c 노드를 현재 Y → 다음 Y 사이에서 보간(interpolate)해서 대각선 배치. 계단 현상 방지.

### Collapse 시퀀스

1. `progress >= COLLAPSE_START` → 트리거
2. 30프레임(~0.5초) 대기 (c 차트 계속 표시)
3. 폭발: scatter + dust 파티클 생성
4. 파티클 소멸 후 결과 텍스트 표시

## 삽질 기록

### 밀도 로직 변경
처음엔 "가치가 높으면 노드 많이 생성"이었으나, 2배 레버리지의 본질(매일 리셋)을 표현 못 함. 당일 수익률 크기 기반으로 변경.

### 튜브 각짐 문제
세그먼트별 색상 적용 시 직선(lineTo)으로 그려서 뚝뚝 끊김. midpoint 기반 quadraticCurveTo + 법선 스무딩으로 해결.

### c 노드 계단 현상
같은 틱의 여러 노드가 동일 Y에 수평 배치 → 계단처럼 보임. 현재 Y와 다음 Y 사이 보간으로 해결.

### 폭발 타이밍
c가 먼저 사라지고 폭발하는 문제. `hasExploded` 플래그로 폭발 전까지 c 차트 유지.

## 핵심 교훈

### 금융 상품의 수학적 구조를 시각화할 때
단순히 "값이 떨어진다"가 아니라, 매일 리셋되는 복리 구조가 왜 위험한지를 체감하게 만드는 게 핵심. 숫자보다 움직임이 직관적.

### 색상은 의미 기반으로
Y 위치 비교로 색을 결정하면 밀도 높은 노드에서 무의미한 색 전환 발생. 데이터(당일 수익률) 기반으로 결정해야 일관성 유지.
