# Day 14 — 1만 명의 가상 한국인

**날짜**: 2026.05.12
**헤드라인**: 엔비디아 Nemotron-Personas-Korea 데이터셋에서 영감받은 가상 한국인 아바타 제너레이티브 아트

---

## 컨셉

엔비디아가 공개한 100만 가상 페르소나 데이터셋(Nemotron-Personas-Korea)에서 영감을 받아, SVG 기반으로 다양한 한국인 아바타를 랜덤 생성하는 인터랙티브 작업. 가로 스크롤로 아바타들을 탐색하고, 클릭하면 각 아바타의 페르소나 정보(이름 + 한 줄 묘사)가 표시된다.

---

## 구현 내용

### 1. SVG 아바타 생성 시스템
- 얼굴형, 헤어스타일, 피부톤, 눈/코/입, 안경, 모자, 귀걸이, 점 등을 파라미터로 랜덤 조합
- 한국인 특성 반영: 피부톤 범위, 헤어스타일(숏컷/투블럭/가르마/단발/장발/포니테일 등)
- SVG 그라데이션으로 깊이감/볼륨감 표현
- 인접 아바타 중복 방지 (피부톤/머리색/옷 색상이 연속으로 같지 않게)

### 2. 캐릭터 다양성
- 남성 헤어 5가지: 숏컷, 투블럭+앞머리, 가르마, 버즈컷, 스파이키
- 여성 헤어 5가지: 단발, 장발, 앞머리+미디엄, 픽시컷, 포니테일/번
- 모자 20%: 야구모자, 비니
- 안경 35%: 둥근/사각/캣아이 + 다양한 프레임 색상
- 귀걸이(여성 60%): 후프/스터드
- 점/주근깨: 랜덤 확률
- 스트라이프 티셔츠: 30% (clipPath로 몸통 안에만 표시)

### 3. 페르소나 시스템
- Nemotron-Personas-Korea 스타일의 한국인 페르소나 랜덤 생성
- 한국 성씨 40개 + 남녀 이름 각 40개 조합
- 직업 30가지 + 나이 범위 + 한 줄 묘사
- 예: '진기태 - 투박한 손마디로 삶의 무게 중심을 잡아온 74세 하역'

### 4. 인터랙션
- 가로 스크롤 (마우스 휠 → 가로 스크롤 변환)
- 클릭 시 아바타 위에 페르소나 정보 팝업
- 호버 시 아바타가 위로 올라가며 밝아지는 반응
- 하단 인터뷰 테이블 (짙은 회색)

## 기술 스택

| 기술 | 용도 |
|------|------|
| SVG (path, ellipse, gradient) | 아바타 렌더링 |
| CSS flexbox + overflow-x | 가로 스크롤 레이아웃 |
| JavaScript | 파라미터 랜덤 생성 + DOM 조작 |
| clipPath | 스트라이프 티셔츠 클리핑 |

---

## 삽질 포인트

1. **머리카락 두피 밀착**: 얼굴이 ellipse로 그려지는데 머리카락 path의 좌표가 타원 상단과 맞지 않아서 빈 공간이 생김. `hairInner`를 `faceTop + faceH * 0.35`로 설정해서 해결.

2. **모자 빈 공간**: 모자 본체와 머리 사이에 빈 공간이 보임. 모자의 하단 좌표를 `faceTop+faceH*0.3`까지 내려서 얼굴과 확실히 겹치게 수정.

3. **인접 아바타 중복**: 랜덤 생성이라 옆에 비슷한 아바타가 나올 수 있음. 이전 아바타의 skinIdx/hairIdx/shirtIdx를 추적해서 다음 아바타는 다른 인덱스를 사용하도록 강제.

4. **스크롤 버벅임**: `wheel` 이벤트에서 `preventDefault` + `deltaY * 2`로 처리하니 버벅임. `deltaY`를 그대로 `scrollLeft`에 더하는 방식으로 변경.

5. **줄무늬 티셔츠 클리핑**: 줄무늬 line이 몸통 path 밖으로 나감. SVG `clipPath`로 몸통 형태 안에만 보이게 수정.

## 동료 공유용 요약

**왜 만들었나**: 엔비디아 Nemotron-Personas-Korea(100만 가상 한국인 페르소나) 데이터셋에서 영감. 다양한 한국인의 모습을 SVG 아바타로 제너레이티브하게 표현.

**핵심 인터랙션**:
- 가로 스크롤로 20명의 다양한 아바타 탐색
- 클릭하면 페르소나 정보 팝업 (이름 + 한 줄 묘사)
- 호버 시 아바타 반응

**기술**: SVG 파라미터 기반 아바타 생성 + CSS 가로 스크롤 + 인접 중복 방지 알고리즘

**삽질**: 머리카락/모자와 얼굴 타원의 좌표 매칭이 가장 어려웠음. ellipse 상단과 path 좌표를 정확히 맞추는 데 여러 번 시도.
