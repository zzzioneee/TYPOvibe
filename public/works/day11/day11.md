# Day 11 — 차이나 쇼크 2.0

**2026.04.30 (수요일)**

FROM · [뉴닉 — 차이나 쇼크 2.0](https://newneek.co/@newneek/article/40314)

## 출발

뉴닉 칼럼에서 "첨단산업을 점령하는 차이나 쇼크 2.0" 기사를 읽고, 중국 오성홍기가 텍스트를 덮으며 펄럭이는 비주얼을 만들고 싶었음. 국기가 텍스트 위를 지나갈 때 글자가 디지털 변형되는 인터랙션이 핵심.

## 구조

- 풀스크린 검정 배경 + 흰 텍스트 "첨단산업을 / 점령하는 / 차이나쇼크 2.0" 좌측정렬
- 오성홍기가 40개 세로 스트립으로 분해되어 S-curve 로 펄럭이며 텍스트 위를 덮음
- 국기와 텍스트가 겹치는 영역에서만 흰/시안 디지털 변형 발생
- 마우스 호버 시 국기 스트립이 크게 들춰짐 (바람 효과)
- 클릭 시 국기가 좌우로 찢어져 텍스트가 드러남 (2.5초 후 닫힘)

## 기술

### 국기 렌더링
- **오성홍기 프로시저 생성** — Canvas 2D 로 공식 규격(30×20 그리드) 기반 별 5개 배치
- **디더링** — 2px 셀 단위로 빨강/노랑 팔레트에서 랜덤 색 할당. 멀리서 국기, 가까이서 픽셀 모자이크
- **S-curve 스트립** — 40개 세로 스트립 × 24개 가로 슬라이스. 각 슬라이스마다 다중 사인파로 좌우 곡선 변위
- **하단 분해** — 스트립 하단 25% 구간에서 슬라이스가 떨어져나감

### 텍스트 겹침 디지털 변형
- **국기 마스크 캔버스** — 매 프레임 국기 스트립을 1/4 해상도 오프스크린 캔버스에 빨강으로 칠하고 `getImageData` 로 마스크 배열 생성
- **텍스트 히트맵** — textCanvas 의 alpha 채널로 텍스트 획 위치 1/0 맵
- **겹침 판정** — `textHitmap[y*W+x] && flagMaskData[...] > 100` 둘 다 true 일 때만 변형
- **변형 효과** — 흰 텍스트 슬라이스를 국기 슬라이스와 동일한 격자로 잘라 약간 밀어냄 + 시안 텍스트 RGB shift 복제
- **시안 텍스트 캔버스** — textCanvas 와 동일한 텍스트를 `#00DDDD` 로 렌더한 별도 캔버스

### 인터랙션
- **마우스 바람 (C)** — 커서 근처 스트립 영향 반경 `flagRW * 0.2`, 변위 강도 `baseAmp * 1.8`. 커서 위/아래에 따라 스트립이 위/아래로 들춰짐
- **클릭 찢기 (B)** — 클릭 위치에서 좌우 스트립이 갈라짐. 0.3초 만에 열리고 2.2초에 걸쳐 닫힘. 최대 벌어짐 화면 8%, 영향 반경 25%

### 성능
- 스트립 40개 × 슬라이스 24개 = 960 drawImage 호출/프레임
- 마스크 캔버스 1/4 해상도 + getImageData 1회/프레임
- DPR 2 지원 (캔버스 해상도 × 2, CSS 크기 원본)

## 핵심 교훈

### 겹침 판정은 마스크 캔버스가 정확
국기가 -8° 회전 + S-curve 변위라 바운딩 박스 근사로는 안 맞음. 오프스크린 캔버스에 국기 영역을 칠하고 getImageData 로 읽는 게 유일하게 정확한 방법. 1/4 해상도면 성능도 OK.

### 텍스트 변형은 국기 위에 그려야 보임
텍스트 변형을 국기 아래에 깔면 국기에 가려져 안 보임. 국기 위에 그리되 textCanvas 가 투명 배경이라 국기를 안 지움.

### fillRect 는 사각형 전체를 칠함
시안 틴트를 fillRect 로 하면 텍스트 형태가 아닌 사각형 전체가 칠해짐. 별도 시안 텍스트 캔버스를 만들어 drawImage 해야 텍스트 획 모양만 보임.

### DPR 2 에서 hitmap 크기 주의
resize 시 W/H 가 DPR 적용된 큰 값으로 바뀌는데, hitmap 빌드 타이밍이 안 맞으면 크기 불일치. fonts.ready + setTimeout 으로 순서 보장 필요.
