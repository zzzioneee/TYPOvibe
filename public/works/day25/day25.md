# Day 25 — SWING Display

**2026.05.29 (목)**

FROM · [참고](https://typovibe.com)

## 출발
타이포그래피를 물리적으로 매달아보면 어떨까? 샹들리에처럼 천장에서 늘어진 도트들이 글자를 이루되, 외부 힘에 반응하는 제너레이티브 타이포그래피.

## 컨셉
물리 시뮬레이션(Matter.js)으로 매달린 도트들이 글자 형태를 유지하다가, 마우스로 흔들면 관성에 의해 출렁이고, 다시 원래 위치로 돌아오는 인터랙티브 디스플레이.

## 구조
- p5.js 캔버스 전체 화면
- Matter.js 물리 엔진으로 각 도트에 constraint 연결
- 마우스 이동 시 주변 도트에 힘 적용
- 글자 레이아웃은 폰트 경로에서 포인트 샘플링

## 기술
### p5.js + Matter.js 연동
- p5.js의 draw loop에서 Matter.js world를 step
- 각 도트가 물리 바디(circle)이며, 원래 위치에 spring constraint로 연결
- stiffness로 복원력 조절 — 값이 낮을수록 더 출렁임

### 텍스트 포인트 샘플링
- opentype.js나 p5.js font.textToPoints()로 글자 외곽선 추출
- 일정 간격으로 포인트를 샘플링하여 도트 위치 결정

## 삽질 기록
- Matter.js의 constraint stiffness 값이 너무 높으면 딱딱하고, 너무 낮으면 형태가 무너짐. 0.02~0.05 사이가 적정.
- 도트 수가 많아지면 물리 연산 부하 → 글자당 포인트 수 제한 필요.

## 핵심 교훈
- 물리 시뮬레이션 + 타이포그래피 조합은 stiffness/damping 파라미터가 결과물의 느낌을 좌우한다.
- 도트 수와 퍼포먼스 사이의 트레이드오프를 초기에 정해야 함.

## 동료 공유용 요약

### 한 줄 요약
샹들리에처럼 매달린 도트가 글자를 이루는 물리 시뮬레이션 타이포그래피.

### 왜 이걸 만들었나
타이포그래피에 물리적 무게감을 부여하면 어떤 인터랙션이 가능할지 실험.

### 작업 흐름
1. p5.js 캔버스 + Matter.js 물리 세계 셋업
2. 글자 외곽선에서 포인트 샘플링
3. 각 포인트에 물리 바디 + spring constraint 연결
4. 마우스 인터랙션으로 힘 적용

### 핵심 인터랙션
- 마우스를 움직이면 주변 도트가 밀려남
- 손을 떼면 spring에 의해 원위치로 복귀
- 출렁이는 과정에서 글자가 일시적으로 변형

### 삽질 포인트
- stiffness 파라미터 튜닝이 전체 느낌을 좌우
- 도트 수 vs 퍼포먼스 밸런스
