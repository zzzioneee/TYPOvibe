# Day 3 — 연애 상대를 찾으러 왜 대형 서점에 갔을까?

**날짜**: 2026.04.20
**헤드라인**: "연애 상대를 찾으러 왜 대형 서점에 갔을까?"
**출처**: [뉴닉](https://newneek.co/@pizzastation/article/39901)

---

## 컨셉

최근 대형 서점이 연애 상대를 찾기 위해 연락처를 묻는 '번따 명소'로 떠오르고 있다는 뉴스에서 영감.
3D 회전 책장에서 책을 고르면 플러팅 문구가 뜨는 인터랙티브 타이포그래피.
서점에서의 설렘과 우연한 만남을 시각적으로 표현.

---

## 구현 내용

### 1. 3D 회전 책장 (CSS perspective + JS)
- CSS `perspective`로 3D 공간 생성
- 50권의 책을 원형으로 배치 (`rotateY` + `translateZ`)
- 스크롤(wheel) + 드래그로 직접 회전 조작
- 관성 적용 — 스크롤 놓으면 서서히 감속
- 마우스 Y 위치에 따라 뷰 앵글 변화 (위 = 하이앵글, 아래 = 정면)

### 2. 책 표지 (실제 출판사 이미지)
- 민음사 세계문학전집 10권
- 문학동네 10권
- 문학과지성사 5권
- 총 25종 × 2 = 50권 배치 (다양성 확보)
- 로컬 이미지로 저장 (CORS 이슈 방지)

### 3. 인터랙션
- **커서 호버**: 책이 위로 플로팅 + 확대 (1.1배)
- **클릭**: 우상단에 선택한 책 표지 + 랜덤 플러팅 문구 표시
- 문구는 다음 책을 클릭할 때까지 유지
- 드래그와 클릭 구분 (이동 거리 5px 이하 = 클릭)

### 4. 플러팅 문구
- 50개 고유 문구 (책마다 다른 문구)
- 서점에서 할 법한 자연스러운 대화체
- 폰트: Noto Serif KR (명조체 — 서점의 문학적 분위기)
- 우측 정렬, 줄바꿈 없이 1줄

### 5. 레이아웃
- 좌상단: 헤드라인 (Pretendard Black, clamp 36~64px)
- 우상단: 선택된 책 표지 + 플러팅 문구
- 중앙: 3D 회전 책장
- 하단: 스크롤 힌트 (← scroll to browse →)
- 전체 올 화이트 배경
- 상단 FROM 링크 → 뉴닉 기사

### 6. 썸네일 처리
- URL 파라미터 `?thumb`으로 썸네일 모드 감지
- 썸네일에서는 헤드라인 타이틀 숨김
- 실제 페이지에서는 정상 표시

## 기술 스택

| 기술 | 용도 |
|------|------|
| CSS perspective + transform | 3D 책장 회전 |
| Vanilla JS | 스크롤/드래그 인터랙션, 관성 물리 |
| Noto Serif KR | 플러팅 문구 폰트 |
| Pretendard Black | 헤드라인 폰트 |

---

## 삽질 포인트

1. **민음사 이미지 CORS**: 외부 URL 직접 로드 시 CORS 차단. 이미지를 로컬에 다운로드해서 해결.

2. **3D 클릭 이벤트**: CSS 3D transform이 적용된 요소에서 마우스 이벤트가 제대로 안 잡힘. 각 slot에 직접 mouseenter/mouseleave/click 이벤트를 걸어서 해결.

3. **드래그 vs 클릭 구분**: 책장 회전을 위한 드래그와 책 선택을 위한 클릭이 충돌. mousedown/mouseup 사이 이동 거리로 구분 (5px 이하 = 클릭).

4. **뷰 앵글 연동**: 마우스 Y 위치에 따라 rotateX를 동적으로 변경. 위로 가면 하이앵글(-20deg), 아래로 가면 정면(5deg).

5. **폰트 결정**: Noto Serif KR ↔ Song Myung 사이에서 고민. 최종적으로 Noto Serif KR의 깔끔한 명조 느낌이 서점 분위기에 더 맞아서 선택.

## 동료 공유용 요약

**왜 만들었나**: 대형 서점이 '번따 명소'로 떠오르고 있다는 뉴스에서 영감. 서점에서의 설렘과 우연한 만남을 3D 책장 인터랙션으로 표현.

**핵심 인터랙션**:
- 스크롤/드래그로 3D 책장을 직접 회전
- 마우스 위치에 따라 뷰 앵글 변화 (하이앵글 ↔ 정면)
- 책을 클릭하면 플러팅 문구가 뜸 ("번호 좀 주시겠어요?", "이 작가 좋아하시나봐요?" 등)

**기술**: CSS 3D perspective + JS 관성 물리 + 실제 출판사 책 표지 50권

**삽질**: 3D transform 위에서 클릭 이벤트 잡기가 까다로웠음. 드래그/클릭 구분은 이동 거리로 해결.
