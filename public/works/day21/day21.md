# Day 21 — 낯선 사람들이 모이면

**2026.05.22 (목요일)**

FROM · [비마이비 — 낯선 사람들이 모이면](https://bemyb.kr/contents/?q=YToxOntzOjEyOiJrZXl3b3JkX3R5cGUiO3M6MzoiYWxsIjt9&bmode=view&idx=170905285&t=board)

## 출발

비마이비에서 "커뮤니티를 정해진 계획대로 굴리는 제품이 아니라, 사람 간의 관계 속에서 끊임없이 변화하고 성장하는 역동적인 유기체" 라는 관점을 읽고, 그 시선 자체를 시각 작업으로 표현하고 싶었음. 텍스트가 '읽히는' 매체가 아니라 '보여지는' 유기체로 존재하는 작업.

## 컨셉

**"낯선 사람들이 모이면 어느 순간 서로가 답이 돼요"** — 이 문장을 조형 활자(pictographic typography)로 표현. 한글 자모를 자연물(줄기, 뿌리, 덩굴, 손, 사과, 새싹, 꽃, 나무 등)로 치환하되, 한글의 초·중·종성 배치 구조는 유지하여 글자가 읽히면서도 그림이 되는 이중 레이어를 만듦.

메타포: **연결 · 연대 · 연속** — 식물이 서로 뻗어 만나듯, 낯선 사람들의 관계가 유기적으로 이어지는 것.

## 구조

- 어두운 사진 배경 (bg.jpg) + 블랙 딤 65%
- 화면 중앙에 4행 텍스트, 가운데정렬
- 각 음절이 자연물 SVG 글리프들의 조합으로 표현됨
- 타이핑 애니메이션: 한 글자씩 리드미컬하게 등장 (빠른 버스트 + 느린 구간 교차)
- 자동 루프 (2.8초 대기 후 리셋)
- FROM 링크 상단 중앙 고정

## 기술

### 조형 활자 시스템 (jamos.jsx)

16개 자모(자음 9 + 모음 7) 각각을 손그림 느낌의 SVG 일러스트로 제작.

- **ㄱ** — 꺾이며 떨어지는 덩굴 (twisting vine)
- **ㄴ** — L자 뿌리 시스템 (root network)
- **ㄷ** — 사람 옆얼굴 (opens to the right)
- **ㄹ** — 꼬인 덩굴 chain (zigzag vine knots)
- **ㅁ** — 네모난 돌/항아리 (solid stone)
- **ㅂ** — 화분 + 두 줄기 (pot with twin stems)
- **ㅅ** — 두 줄기가 만나는 나무 (V-apex tree)
- **ㅇ** — 사과 (solid apple with stem + leaf)
- **ㅊ** — 꽃 (bud + horizontal leaf bar + twin roots)
- **ㅏ** — 세로 잎 + 오른쪽 가로 가지
- **ㅓ** — 세로 잎 + 왼쪽 가로 가지 (mirror)
- **ㅗ** — 가로 땅 + 위로 솟는 줄기 (flower/sun)
- **ㅜ** — 가로 캐노피 + 아래로 매달린 열매
- **ㅡ** — 애벌레 (horizontal caterpillar)
- **ㅣ** — 야자수 (tall palm trunk)
- **ㅕ** — 수직 줄기 + 왼쪽 두 잎 가지

추가 글리프: **ㅐ** (두 기둥 사이 얼굴), **ㅛ** (땅 + 두 새싹)

### SVG 필터 — 손떨림 + 노이즈 텍스쳐

```
hd-wobble: feTurbulence(0.035/0.06) → feDisplacementMap(5.5) → feTurbulence(0.18) → feDisplacementMap(2.2)
hd-wobble-soft: feTurbulence(0.05) → feDisplacementMap(3)
graffiti-noise: feTurbulence(1.2) → feComponentTransfer(discrete) → feComposite(in)
```

- **hd-wobble** — 2단계 displacement로 윤곽이 삐뚤빼뚤하게 흔들림. 레퍼런스의 "사람이 그린 듯한 조악한 표현" 구현
- **graffiti-noise** — 스프레이 페인트/벽면 그래피티처럼 채움 면에 구멍이 뚫린 질감

### 한글 조합 레이아웃 (syllables.jsx)

각 음절의 초·중·종성을 position:absolute로 겹쳐 배치. 4가지 레이아웃 타입:

- **V** (세로 모음, 받침 없음): 초성 왼쪽 | 중성 오른쪽 — 사, 가, 이 등
- **H** (가로 모음, 받침 없음): 초성 위 / 중성 아래 — 모, 로, 요 등
- **VB** (세로 모음 + 받침): 초중성 위 / 종성 아래 — 낯, 선, 람 등
- **HB** (가로 모음 + 받침): 초성 / 중성 / 종성 세로 스택 — 들, 순 등
- **DOE** (커스텀): '돼' — ㄷ(좌상) + ㅗ(하단) + ㅐ(우측) 복합 배치

각 자모 박스가 의도적으로 OVERSIZED (100% 넘는 width/height) → 글리프들이 겹치면서 유기적으로 연결된 느낌.

### 타이핑 애니메이션 (app.jsx)

- **리드미컬 속도 변화**: 5글자 주기로 3글자 빠르게(80~120ms) + 2글자 느리게(280~400ms)
- **자동 루프**: 전부 표시 후 2.8초 대기 → 리셋
- **즉시 등장**: opacity/transform transition 없이 선명하게 팝인

### 클로드 디자인 활용

초기에 직접 SVG path를 작성했으나 글리프가 너무 추상적이고 읽히지 않았음. 클로드 디자인(Claude Design)으로 만든 프로토타입이 훨씬 명확하여 해당 버전을 채택. 주요 장점:
- 면(silhouette) + 라인(outline) 혼합으로 형태감 확보
- feTurbulence displacement로 손맛 자동 적용
- React + Babel in-browser로 빠른 프로토타이핑

## 삽질 기록

### v1: 자모 분해 방식 (실패)
처음엔 한글을 자모 단위로 분해하여 'ㄴ ㅏ ㅊ ㅅ ㅓ ㄴ…' 식으로 나열. 당연히 읽히지 않음. 글자 단위 배치로 전면 수정.

### v2: 직접 SVG path 작성 (실패)
viewBox 60×72 안에 path를 직접 그렸는데, 추상 기호처럼 보여서 "뭘 그린 건지" 안 느껴짐. 줄기인지 뿌리인지 구분 불가.

### v3: 클로드 디자인 버전 채택
클로드 디자인으로 만든 프로토타입이 레퍼런스(상형문자 스타일 조형 활자)에 훨씬 가까웠음. 그대로 가져와서 index.html로 교체.

### v4: 배경/색상 전환
원래 크림색 종이 배경 + 검정 잉크였으나, 제공된 bg.jpg를 배경으로 하면서 어두운 톤으로 전환. 글리프를 흰색으로, graffiti-noise 필터로 스프레이 질감 추가.

### v5: '돼' 글자 처리
'돼'는 ㄷ+ㅙ(ㅗ+ㅐ) 복합모음. 처음에 H타입(ㄷ+ㅗ)으로만 처리 → '도'로만 보임. 커스텀 DOE 레이아웃으로 ㅐ까지 포함.

### v6: 타이핑 리듬
일정 간격(380ms) → ±30% 랜덤 → 5글자 주기 버스트 패턴으로 점진 개선. 빠른/느린 대비를 높여야 체감됨.

## 핵심 교훈

### SVG 필터로 손맛 내기
직접 불규칙 path를 그리는 것보다 feTurbulence displacement가 훨씬 효과적. 깔끔한 path + 필터 = 일관되면서도 손그림 느낌.

### 조형 활자에서 '읽힘'의 핵심은 구조 유지
글리프 자체가 아무리 추상적이어도, 한글 초중종성의 상대적 위치만 유지하면 뇌가 글자로 인식함. 위치 구조가 깨지면(자모 나열) 절대 읽히지 않음.

### 면(fill)이 형태 인식의 핵심
라인만으로는 "이게 뭐지?" 상태. 실루엣(면 채움)이 있어야 "아, 사과구나" "나무구나" 즉시 인식됨. 레퍼런스도 면 위주.

### 타이핑 리듬은 대비가 핵심
균일한 랜덤(±30%)은 사람 눈에 "그냥 일정" 으로 느껴짐. 빠른 구간과 느린 구간의 차이를 2배 이상 벌려야 체감됨.

## 동료 공유용 요약

### 한 줄 요약
한글 자모를 자연물(덩굴·사과·나무·꽃 등)로 치환한 조형 활자가 타이핑되듯 한 글자씩 등장하는 인터랙티브 타이포그래피.

### 왜 이걸 만들었나
비마이비 글에서 "커뮤니티는 유기체"라는 관점을 읽고, 텍스트 자체를 유기체(자연물)로 만들면 어떨까 싶었음. 상형문자처럼 그림이 곧 글자가 되는 경험.

### 작업 흐름
1. 레퍼런스 분석 → 면+라인 혼합 상형문자 스타일 확인
2. 직접 SVG path 시도 → 실패 (너무 추상적)
3. 클로드 디자인 프로토타입 채택 → SVG filter로 손맛 적용
4. 한글 조합 레이아웃(V/H/VB/HB) 구현 → 글자로 읽힘
5. 타이핑 리듬 패턴(버스트+슬로우) 적용
6. 어두운 사진 배경 + 흰색 글리프 + graffiti 노이즈

### 핵심 인터랙션
- **타이핑 등장**: 한 글자씩 팝인. 빠른 버스트(80~120ms) + 느린 구간(280~400ms) 교차.
- **자동 루프**: 전부 등장 후 2.8초 대기 → 초기화 → 반복.
- **SVG 워블**: feTurbulence displacement로 모든 글리프가 삐뚤빼뚤하게 흔들림.

### 삽질 포인트
- 자모 단위 분해 → 글자 단위 배치로 전면 수정 (읽히지 않음)
- 직접 path 작성 → 형태감 부족으로 실패, 클로드 디자인 버전이 훨씬 나았음
- '돼' 복합모음(ㅙ) 처리 → 커스텀 레이아웃으로 해결
